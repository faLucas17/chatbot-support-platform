PARTIE 2 : API ENDPOINTS (À tester sur Postman)
================================================================================

ENDPOINTS DE BASE :
--------------------------------------------------------------------------------

1. POST /api/message/
   Description : Envoyer un message et recevoir une réponse du bot
   Body : {
       "api_key": "31d4efff-2b32-481f-a17f-d2017982f7ed",
       "conversation_id": null,
       "content": "Votre question ici"
   }

2. GET /api/conversation/<id>/
   Description : Récupérer l'historique d'une conversation
   Query params : api_key=...

3. POST /api/agent/reply/
   Description : L'agent répond à une conversation
   Body : {
       "api_key": "31d4efff-2b32-481f-a17f-d2017982f7ed",
       "conversation_id": 1,
       "content": "Réponse de l'agent"
   }

4. GET /admin/conversations/
   Description : Liste de toutes les conversations

5. POST /admin/conversations/<id>/
   Description : Répondre à une conversation (admin)

6. GET /api/user/conversations/
   Description : Liste des conversations de l'utilisateur connecté

ENDPOINTS POUR DOCUMENTS :
--------------------------------------------------------------------------------

7. POST /api/documents/upload/
   Description : Uploader un document (PDF, TXT, image)
   Body (form-data) : {
       "api_key": "31d4efff-2b32-481f-a17f-d2017982f7ed",
       "title": "Nom du document",
       "file": (fichier à uploader)
   }

8. GET /api/documents/
   Description : Liste des documents uploadés
   Query params : api_key=...

9. DELETE /api/documents/<id>/
   Description : Supprimer un document
   Body : { "api_key": "..." }
